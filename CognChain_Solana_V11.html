02 — The Primitive

The Memory Record Primitive

Not model weights. Not raw logs. A compact memory artifact built for Solana settlement: ownership, hash, policy, score and checkpoint.

schema · SolanaMemoryRecordv0.5
// A CognChain record is a portable memory object —
// reproducible, permissioned, and anchored on Solana.

// Agent model discriminator — stored as u8 enum, no heap allocation
enum AgentType { Claude=0, GPT=1, Gemini=2, Custom=254, Open=255 }

struct SolanaMemoryRecord {
  // Anchor discriminator — 8 bytes, auto-prefixed by the framework
  // discriminator: [u8;8]  // sha256("account:SolanaMemoryRecord")[..8]

  // Ownership & identity — all fixed-size, no heap
  Vault:             Pubkey      // user memory vault PDA
  MemoryId:          u64         // monotonic id inside the vault
  WriterAgent:       Pubkey      // delegated agent or user authority

  // Content hashes — crystallized reasoning, never raw text on-chain
  ContentHash:       [u8;32]    // canonical hash of normalized payload
  SummaryHash:       [u8;32]    // hash of approved summary
  SourceHash:        [u8;32]    // hash of source bundle or trace set

  // Quality signals — all fixed-size integers, cheap on Solana accounts
  PolicyId:          u16         // permission / retention policy
  ConfidenceBps:     u16         // normalized confidence (basis points)
  ImportanceBps:     u16         // importance score (basis points)

  // Provenance — fixed-size to keep account rent predictable
  ExtractionVersion: [u8;8]     // crystallizer version as UTF-8 bytes (e.g. b"v0.5.0\0\0")
  SourceAgentType:   AgentType  // u8 discriminant, no dynamic allocation
  CreatedAt:         i64         // Unix timestamp (Clock::get().unix_timestamp)
}

03 — Verification

How Memory Gets Verified

CognChain does not replace Solana consensus. Solana orders and settles transactions; CognChain adds a verification pipeline above it that decides what deserves to become portable memory.

The Verification Score

quality_score =
task_success_rate×0.5blind benchmark
cross_agent_reuse×0.3% agents that adopted it
temporal_consistency×0.2rolling 7 / 30 / 90d
How each variable is measured: task_success_rate — dual signal: explicit user thumbs-up/down on agent output + the agent's own outcome verification (did the tool call succeed? did the follow-up prompt confirm the result?). cross_agent_reuse — counted by the off-chain indexer from Anchor event logs: every time a vault imports a record via the permission grant instruction, the program emits a MemoryImported event; the indexer aggregates distinct vault pubkeys per record. This avoids mutable on-chain counters that would require account realloc as reuse grows. temporal_consistency — rolling average of per-agent reuse events over 7, 30 and 90-day windows; records that stop being reused decay in score. All three are computed off-chain and submitted with the memory write; the on-chain program validates that the submitted ConfidenceBps is within the acceptable policy range before settling.

Why This Category Matters

What the ZK Circuit Proves

The Groth16 circuit for Tier-2 memory records proves three statements simultaneously without revealing memory content: (1) Authorship — the ContentHash was produced by an agent holding the authority key for the target vault PDA. (2) Quality threshold — the ConfidenceBps and ImportanceBps values exceed the policy-defined acceptance thresholds, computed over the private payload. (3) Policy compliance — the PolicyId referenced on-chain was satisfied by the private reasoning trace. The Solana verifier program checks the proof and the public inputs (vault PDA, content hash, policy id) in a single transaction. Raw reasoning, session content and embeddings never touch the chain.

Solana Program Layer

CognChain Core — Anchor Program

Anchor account constraints enforce ownership and mutation rules at the protocol level, not the application level. Key constraints: seeds = [b"vault", user.key().as_ref()] for deterministic PDA derivation; has_one = authority on every write instruction to prevent unauthorized mutations; bump stored in account to avoid brute-force re-derivation on every call. Agent delegation uses a separate PermissionGrant PDA scoped to (vault, agent_pubkey, policy_id) — revoking an agent closes the grant account and reclaims rent. The program is upgradeable during testnet and early mainnet; upgrade authority is held by a Squads Protocol multisig requiring team + advisor co-signature. Upgrade authority will be transferred to a DAO or burned once the account model is battle-tested.

06 — Business Model

Revenue Before Token

The business model comes from useful infrastructure first. API usage, enterprise subscriptions and settlement fees build the foundation before any token economics are considered.

Solana Cost Model per Memory Write

A single procedural memory record on Solana costs approximately ~0.0025 SOL as a one-time rent-exemption deposit for a 256-byte account (hashes + scores + metadata, no raw content). Accounts are rent-exempt by design — the deposit is paid once at creation and the account persists indefinitely; there is no recurring rent. Compute is ~5,000–8,000 CUs per write instruction — well within Solana's 1.4M CU limit per transaction. At current SOL prices, this is sub-$0.01 per memory record. The economics are viable at scale without any token layer.
